Como o Qase deve ser organizado

Esta página define a estrutura oficial do Qase necessária para que o WellnessQAReporter funcione corretamente.

Ela serve como documento-pai para as subpáginas que detalham cada elemento (Suites, Cases, Plans, Runs e Resultados).

Objetivo:

Garantir que as equipes utilizem o Qase de forma padronizada, consistente e alinhada entre todos os projetos (Fully e Chubb), permitindo que o Reporter colete, interprete e consolide os dados de maneira precisa.

Por que a organização do Qase é essencial?

  • Define o padrão que garante a integridade dos KPIs.
  • Evita divergências entre plataformas (iOS e Android).
  • Permite diferenciar corretamente dados por idioma (PT/EN/ES).
  • Sustenta rastreamento entre Suite → Caso → Execução → Resultado → Defeito.
  • Reduz retrabalho e melhora a governança do QA.

Estrutura Oficial do Qase:

A organização deve seguir quatro pilares principais:

1. Suites organizadas por funcionalidade

Cada suite representa uma área funcional do aplicativo. Isso permite:

  • Relatórios por módulo
  • Análise de impacto por funcionalidade
  • Agrupamento correto dos casos no Reporter

2. Casos de teste completos e atualizados

Cada caso deve conter:

  • título padronizado
  • descrição clara
  • pré-condições
  • passos detalhados
  • resultado esperado
  • tags de plataforma (iOS/Android)
  • tags de idioma (PT/EN/ES), quando aplicável

3. Plans por Release Quinzenal

Cada ciclo de release deve ter um plan dedicado, contendo todas as suites relevantes.

Nomes devem seguir padrão claro para facilitar a leitura pelo Reporter.

4. Runs novas e completas

Toda execução deve gerar uma run nova por plataforma, nunca reaproveitada.

Execuções incompletas ou sem resultados comprometem diretamente os KPIs.

Subpáginas Relacionadas:

  • Suites & Cases – Como organizar funcionalidades e casos
  • Plans – Como estruturar releases quinzenais
  • Runs – Como registrar execuções corretamente
  • Registro de Resultados – Como marcar, documentar e evidenciar

Conclusão:

Seguir este padrão garante dados limpos, rastreáveis e prontos para análise, permitindo que o WellnessQAReporter gere KPIs de alta confiabilidade e reduza o esforço manual dos times.
